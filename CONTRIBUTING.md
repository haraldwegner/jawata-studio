# Contributing to GOJA Studio (goja-studio)

Thanks for your interest in GOJA Studio — the desktop control plane (Tauri + Rust + Svelte)
that manages GOJA workspaces and deploys the MCP server into your agent's config. A few ground
rules keep the project sustainable and safe to change.

## Licence & CLA

goja-studio is licensed under **AGPL-3.0** (see [`LICENSE`](LICENSE) / [`NOTICE`](NOTICE)).

By contributing you agree to the **GOJA Contributor License Agreement** ([`CLA.md`](CLA.md)),
recorded automatically via **CLA-assistant** on your first pull request. The CLA lets the
maintainer keep the project sustainably licensed (including any future dual-licensing) while
**you retain full rights to your own work**.

## Provenance — sign your commits

Every commit must carry a Developer Certificate of Origin sign-off:

```bash
git commit -s -m "feat(scope): your change"
```

This certifies you wrote the change, or have the right to submit it.

## Attribution policy

Commits are attributed to their **human author only**. Do **not** add co-author or
generated-by trailers for AI / code-generation tools — e.g. no `Co-Authored-By:` lines and
no "Generated by …" notes in commits, PR bodies, code comments, docs, or release notes.
PRs that carry such trailers will be asked to remove them before merge.

## Building

```bash
# Rust backend
cd src-tauri && cargo build && cargo test
# Frontend
npm install && npm run build
# Full desktop app (dev)
npm run tauri dev
```

## What we look for

- Build green (`cargo test` + `npm run build`) and the touched tests pass.
- **Small, reviewable PRs** — one logical change each.
- Behaviour-preserving changes; isolate any behaviour change in its own tested commit.

## Security

Report vulnerabilities **privately** per [`SECURITY.md`](SECURITY.md). Please don't open
public issues for security bugs.
