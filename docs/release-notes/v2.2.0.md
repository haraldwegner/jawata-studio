# GOJA Studio v2.2.0 — the Memory view, done right (Sprint 21b)

Pairs with goja-mcp v2.2.x (autofind, skip-unchanged loads, shared-store hardening).
Absorbs the unreleased v2.1.1 fixes.

## Memory view — goals in the UI, verbs at the prompt

The v2.1.0 Knowledge pane mirrored the full 15-verb API and was rebuilt from user
feedback into two panels + results:

- **Memory sources** — auto-discovery is the default (layered CLAUDE.md, every
  `~/.claude/projects/*/memory`, Cursor rules, `AGENTS.md`, Copilot instructions —
  resident ≥ v2.2.0); extra roots via a folder picker + removable list. Store mode:
  `shared` (default) or `workspace`.
- **Store & Maintenance** — ONE row per store (a shared store is one store, not one row
  per workspace), entries + size + location; actions: **Load / Clean up / Export… /
  Import… / Wipe**, with export/import through the OS file dialogs. `Load` fans out to
  every reachable resident of a shared store (each contributes its own project
  locations). Results render human-readable directly below the actions, raw JSON behind
  a details fold. Tooltips on every control carry the prompt-phrase vocabulary.
- Removed: the curation panel, the backups/GC panel, crawl-cap knobs, the `memory`
  store-mode — curation and staleness refresh run by prompt/automatically; backup GC
  runs automatically at manager startup.

## Responsiveness & robustness

- **The app no longer freezes while residents boot** — the Memory-view commands were
  synchronous Tauri commands (main-thread!) doing seconds of HTTP; they are now async
  with the HTTP off-thread.
- Unreachable residents are re-polled every 5 s for as long as the view is open
  ("resident starting — retrying automatically…"); actions re-enable by themselves.
- Scattered legacy `.bak` files are swept into the managed backup area automatically
  once per manager start (recognized patterns only; report logged).

## Also

- Platform-exact release-asset selection (previously the first `.tar.gz` — alphabetical
  — could install the wrong architecture; v2.1.1 fix, first shipped here).
- Settings pruned: crawl caps and the recursive toggle are gone (the resident's
  defaults are runaway backstops; the crawl finds everything).
- In-app Help gained a **Memory** section.
