# jawata-studio v3.3.0 — the injector's client side (Sprint 26a)

The client half of the redesigned intelligent injector (see jawata-mcp v3.3.0).

## The guard is reflex-aware

The `.java`-edit guard already blocks a hand-edit and points to the refactor
tools. It now also reads the edit's intent: a hand-rolled timer
(`nanoTime` / `currentTimeMillis` / `Stopwatch`) is met with **use `profile` —
it samples the running JVM and names the hotspot as a symbol, zero code
change**; debug-armor (`System.out/err.print`, `printStackTrace`,
`logger.debug/trace`) is met with **use `debug` — attach and set
`probe_set kind=logpoint`, zero code change**.

This is a smarter message on an edit already blocked — **not** a new block. An
ordinary `logger.info` does not trigger it (no false positives on real logging).

## The seat workflow is coded

The rule block deployed into every client's rule file now names **when each
seat fires in the dev process**, by rule (not left to the agent remembering):
the architect at the design step and at every checkpoint (a watch-diff), the
test-writer at the coverage gate, the javadoc-writer at the doc gate, the
debugger/profiler at the runtime reflexes.
