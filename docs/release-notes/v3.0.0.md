# jawata-studio v3.0.0 — the runner: jawata drives its own agents

**The v3.0 milestone (Sprint 25).** The manager becomes a control plane for
narrow autonomous agents. The runner spawns a seat (a markdown-defined agent:
model tier, effort, schedule, gate classes, ceilings, stance), drives it
through a bounded **detect → do → verify → stop** loop over a pluggable
model-CLI adapter (Claude Code headless first), and refuses to land anything
that fails its gates. **Propose mode throughout: a human approves every
landing.**

## What's in the runner

- **Seats**: `seats/*.md` — javadoc-writer, test-writer, architect (design +
  watch modes), debugger, profiler, spec-editor, spec-auditor. Malformed
  definitions error loudly; model tiers are set by a measured tiering table
  (same fixtures, same gates, small vs strong).
- **Gates**: purity (scope containment) + compile-verify always; behavior /
  tests (pass ×3, coverage delta, mutation-bite) / docs (doclint) per seat.
  Gates judge the PROPOSED state on a shadow copy; a gate that cannot run has
  NOT passed; refusals keep their evidence record.
- **Containment**: wall TTL, iteration and cost ceilings — a breached run is
  reaped with its whole process tree and journaled as `reaped: <ceiling>`.
- **The journal**: every run (including "nothing to do") appends one labeled
  jsonl row — seat, target, evidence, gates, verdict, human verdict, outcome,
  cost. This is the corpus later sprints learn from.
- **Proposals**: whole-file blocks (model-emitted diffs proved unreliable);
  the runner computes the record diff deterministically; each record ships
  `proposal.md`, `diff.patch`, `gates.json` and the staged files.
- **Scheduler**: cron-subset `schedule:` fields fire seats on cadence.
- **Design mode**: the architect produces target-architecture artifacts with
  a mandatory picture (`ARCHITECTURE-<scope>.md`) — the runner subsystem's
  own picture ships in this repo — and judges later diffs against them.
- **Injected communication contract**: the managed rule block deployed into
  client configs now carries the upward-communication contract
  (decision-first asks, options with pros/cons, one decision per ask,
  abbreviations defined at first use).

Every seat was proven LIVE before this release: real model CLI, sandbox
resident on the deployed jawata jar, gates green — including a test-writer
run that took a fixture from 1 to 17 passing tests and its mutation kill
count from 6 to 14, and a debugger run whose one conditional breakpoint
positively proved a seeded boundary bug. Evidence in the sprint-25 dossier.

Pairs with jawata-mcp 3.0.0 (async family sweeps; the same 45 tools).
