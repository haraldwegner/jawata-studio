# goja-studio v2.0.0 — Knowledge push channel (Sprint 21, the "learn" capstone)

The delivery half of GOJA's knowledge store. goja-mcp v2.0 *holds* the store; goja-studio v2.0
**pushes** its knowledge into the agent's context at the right moment, through the same hook channel
Sprint 22 shipped. A frozen model can't learn between sessions — so the store is **push-memory**:
the harness injects the relevant knowledge, per layer. Coordinated suite 2.0 with goja-mcp v2.0.

## Two managed hooks — per-layer push

- **SessionStart domain primer** (`sessionstart-primer.sh`) — the always-on layer. At session start it
  live-calls `experience(kind=primer, format=text)` and injects the domain nodes (what this codebase
  is *about*) as session context — the agent starts oriented, before it touches anything.
- **PreToolUse cue-gated recall** (`pretooluse-recall.sh`) — the on-demand layer. Before a GOJA
  refactor it extracts the target symbol, calls `experience(kind=recall, format=text)`, and injects the
  **terminal** node (prior hazards / lessons / failure modes) — or stays silent on absence. A separate
  PreToolUse entry that **coexists with the enforcement guard**.

## Robust by construction

- Both scripts bake the resident `/mcp` URL + Bearer token (like the guard's `health_url`) and call the
  server directly — POST `/mcp` returns the JSON-RPC result in the body, no handshake.
- **Rendering lives in the mcp** (`experience(..., format=text)`, compiler-tested + sanitized), so the
  hooks only peel goja's fixed MCP envelope — they never parse variable tool structure.
- **Fail-safe by design**: goja down / no cue / empty / absence / any parse miss → inject nothing, so
  the session and the tool call proceed exactly as before. The knowledge push is purely additive.

## Anti-RAG delivery

The push never dumps a pile: the fit-gated terminal contract means the model sees **one fitting node or
nothing** — the injection *shrinks* the working context (resolve or dismiss), it never enlarges it.

## Under the hood

A generic `write/remove_managed_hook_section` now backs the SessionStart + PreToolUse push hooks
without touching the shipped guard/observer; wired into the deploy + remove flows; idempotent +
byte-stable. `cargo test` green.
