# goja-studio v2.3.0 — The prompt-boundary recall (Sprint 21c, item D)

The missing middle link of the memory loop ships: *get the discussion → make
keywords → look up in the store → retrieve and use as context.*

## New managed hook: `userpromptsubmit-recall.sh`

On every user prompt in Claude Code, the hook extracts content-bearing keyword cues
(stopword-stripped, at least 2 content tokens; the best trigram gets one attempt,
bigrams get two), asks the resident's experience store terminally, and injects the
ONE fitting atomic fact as context before the turn begins. Ambiguous answers (2+
fitting facts) try the next cue instead of guessing; absence stays silent; slash
commands and trivial prompts are skipped; every failure path is a silent `exit 0` —
the session is never blocked or slowed (per-attempt timeout 2 s, local answers in
milliseconds).

Managed exactly like the SessionStart primer and the PreToolUse refactor-recall:
installed and updated on deploy with the resident URL + token baked in, verified by
the deploy selftest (which shares the live emit format), removed cleanly on
undeploy, and always preserving your own hooks in `settings.json`.

Pairs with goja-mcp v2.3.0 (atomic facts + keyword harvest): the enriched corpus is
what makes prompt-derived cues actually hit.
