# goja-studio v1.3.1

A dogfood fix for the v1.3.0 escape valve: the declared-fallback **reason is now
captured cleanly** in the audit log.

## Fix — clean reason capture in `fallback.log`

v1.3.0 extracted the `goja-fallback: <reason>` text from the *flattened* tool input,
so it greedily swallowed everything after the marker — a multi-line command's other
lines, and trailing JSON punctuation (`"}}`) — bounded only by a 200-char cap.

v1.3.1 captures **only the reason on the marker's own line**: it reads from the
un-flattened input and trims at the first backslash (a JSON escape / `\n`) or
double-quote (the JSON string close), then strips trailing space and caps length.

Before → after for a multi-line command:

```
scanning comments\necho done\nls -la"}}      (v1.3.0 — polluted)
scanning comments                            (v1.3.1 — clean)
```

Why it matters beyond tidiness: `~/.claude/goja-studio/fallback.log` is the audit
trail of deliberate fallbacks — and it's the training signal a future
intelligent-injector policy learns from. A clean reason field keeps that signal
honest at the source.

No behavior change to the guard's block/allow decisions.
