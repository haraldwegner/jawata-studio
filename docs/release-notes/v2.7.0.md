# jawata-studio v2.7.0 — the jawata release

goja-studio is now **jawata-studio**. Same manager, new trademark-clean name, new home:
**[jawata.org](https://jawata.org)** — and the new arch mark in the app icon and tray.

## Automatic migration — nothing lost

First launch migrates your existing setup in place:

- **App dirs move once:** `~/.config|~/.cache|~/.local/state/goja-studio` →
  `…/jawata-studio`; the Claude hooks dir `~/.claude/goja-studio` →
  `~/.claude/jawata-studio`. Older generations (`javalens-manager`) are left untouched —
  one move per generation, never a clobber.
- **Knowledge store moves as-is** (`~/.local/share/goja` → `~/.local/share/jawata`) and
  the resident rewrites its symbol anchors to the new `org.jawata.*` names (jawata-mcp
  v2.7.0 schema v4) — every note stays recallable.
- **Agent wiring renames itself on deploy:** MCP server keys become `jawata-*` and stale
  `goja-*` generations are stripped; Claude/Cursor hooks, rules and scripts are rewritten
  to their `jawata` names (old rule files superseded, never deleted).
- Saved settings self-heal: legacy release-repo slugs and the `GOJA_RELEASE_REPO`
  environment variable are still recognized for **one release**, then removed.

## Changed

- Release source now targets `haraldwegner/jawata-mcp`. The defunct upstream
  (`pzalutski-pixel/javalens-mcp`) download option is **removed** — it pointed at a repo
  that no longer serves compatible releases.
- Runtime UI: "GOJA Runtime" is now "JAWATA Runtime" throughout.

## Fixed

- Local-JAR runtime source could not be saved (`missing field jar_path`) — the
  camelCase wire format of the `localJar` source was rejected. Latent since v1.0.0.
- A plain "Deploy to Agents" now strips stale managed MCP server generations; before,
  outdated managed keys survived in `~/.claude.json` / `~/.cursor/mcp.json` until a
  forced regenerate.

## Before upgrading

Quit the old goja-studio **and make sure no old `goja.jar` residents are still
running** (`pgrep -af goja.jar`) before first launch — a surviving old resident keeps
the store and ports busy, which can block workspace start and leave the automatic
pre-migration backup empty. jawata-studio v2.7.1 will handle this automatically.
