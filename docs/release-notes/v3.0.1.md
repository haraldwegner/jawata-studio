# jawata-studio v3.0.1 — dogfood + audit patch on v3.0.0

The standing rule applied to the runner release: dogfood-in-anger plus a
fresh-context implementation audit of the released code against the
deliverable bodies. One real dogfood catch and four audit cures.

## Fixed

- **Deleting a workspace now asks first.** `delete_workspace` wipes the
  workspace fully — its running service, every project, all analysis data —
  and its control sits next to the deploy controls, so it was mis-hit when
  the intent was "redeploy" (caught live, repeatedly). Every delete path now
  confirms with the consequences named; "Delete all" gets one strong
  confirmation for the whole wipe.
- **The profiler seat carries the full diagnosis discipline.** "Enumerate
  the exits" and "recall before you theorize" were missing (the audit's
  point: the discipline is seven elements, not five-minus-two); the harness
  now recalls the profiled symbols from the experience store into the
  evidence. Re-proven live against the deployed 3.0.0 runtime.
- **A fake-passing gate is now a loud refusal.** The generic Tests-class
  `coverage_delta` gate passed on any successful tool response without
  verifying a delta; a gate that cannot verify has NOT passed — it now
  fails with the reason (test-writing seats run the full TestWriterGates
  chain, which verifies for real).
- **Records now match proven behavior**: the javadoc-writer stance no longer
  demands per-type runs its harness never did (one grouped batch is the
  proven contract), and the spec-editor/spec-auditor tier fields are marked
  policy-set (unmeasured) instead of implying a measured table.
- Live-gate tests discover the deployed jawata jar version-agnostically
  (the fleet flip to 3.0.0 broke a hardcoded path).
