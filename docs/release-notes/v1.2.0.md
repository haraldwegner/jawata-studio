# goja-studio v1.2.0

The desktop companion to **goja-mcp v1.5.0** (multi-step orchestration). This
release is the **usage layer** of Sprint 18 Track 2 — it makes the agent actually
*use* GOJA, and makes a multi-workspace setup safe for subagents.

## Highlights

- **Health-gated ASK (rule block).** The deployed rule block now tells the agent:
  on Java work, if GOJA is unreachable, **stop and ask** how to proceed — never
  silently fall back to grep. Non-Java work (Rust, Python, configs) is unaffected,
  and it carries an anti-dodge guard so the agent can't reclassify Java work to
  skip the question.

- **PreToolUse enforcement hook (Claude Code).** Deploy now writes a managed
  `PreToolUse` hook into `~/.claude/settings.json` that redirects Java **text
  search** (grep/rg/find/sed/awk, or the Grep tool, over `*.java`) to GOJA's
  compiler-accurate tools. It is **health-gated**: when GOJA is up it redirects to
  `search_symbols` / `find_references` / `analyze`; when it's down it blocks with a
  diagnosis and how to start. Non-Java calls and file edits pass untouched — the
  hook enforces only the unambiguous, high-precision case, so it can never block a
  legitimate edit. Other clients keep the rule block. The hook is written on deploy,
  removed on delete, and preserves any user-authored hooks.

- **Deterministic gateway routing — the subagent mis-route is fixed.** A
  `tools/call` with no `filePath`/`projectKey` used to fall through to an arbitrary
  first resident, which is why subagents (that omit the locator) silently landed in
  the wrong/empty workspace. The gateway now **auto-routes a single workspace** but,
  for **more than one**, returns a clear error naming the valid `projectKey`s
  instead of guessing. A subagent that omits `projectKey` now fails safe and
  self-corrects.

- **Help rewrite.** The in-app help now explains **how GOJA works** — the
  detect → goal → how loop, the parity-gated `refactoring(action=plan)`
  orchestration, reuse-over-reinvent primitives, and the health-gated fallback —
  and **defers the tool list to Settings → Exposed Services → Test Services**, the
  live authoritative surface. No more hand-maintained enumeration to drift.
