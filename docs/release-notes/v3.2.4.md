# jawata-studio v3.2.4 — themed confirmation dialog

A polish patch on top of v3.2.3's delete-guard fix.

## The look-and-feel fix

v3.2.3 made destructive confirmations actually appear by routing them through
the OS-native `tauri-plugin-dialog`. That renders reliably, but as an unstyled
GTK/OS message box that does not match the app.

v3.2.4 replaces it with an in-app **Svelte modal** (`ConfirmDialog`) that both
renders reliably — it is app DOM, not a browser or OS API — and matches the
app's dark theme (the same panel, backdrop, and buttons as the rest of the
UI). Every destructive action (delete workspace, Delete all, memory
clean/wipe, runtime clean, quit-with-services) uses it through the one
`confirmDestructive()` helper.

Behavior: Cancel, Esc, or clicking the backdrop aborts. There is deliberately
no Enter-to-confirm — for a destructive action, OK must be an explicit click,
so a stray keypress can never delete anything.

No engine (jawata-mcp) change.
