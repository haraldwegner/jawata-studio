# jawata-studio v3.3.1 — /sprint ships, /train retires

## /sprint is a real command now

The two-seat sprint pipeline — EDITOR writes, a fresh-context AUDITOR can
REFUSE and loops until sign-off, the user signs off last — was the **one command
jawata never shipped**. It lived hand-placed in a single machine's
`~/.claude/skills/`, so the process the whole workflow depends on was simply
absent on every other client and machine. It surfaced the obvious way: opening
a second machine and finding no `/sprint`.

It is now single-sourced in the repo (`skills/sprint.md`) and deploys exactly
like the seats — Claude, Cursor, Antigravity, and the claude.ai bundle. Its
tests assert the **pipeline** arrived (the auditor seat, the refuse-loop, the
RAW baseline), not merely that a file was written.

## /train is retired

Sprint 26a deleted every ML model class, so the command had nothing left to
train: its entire output had become a "retired" tombstone. A command whose
whole content is a tombstone is noise in every client that ships it. It is
removed from the deploy set, and the tool kinds behind it are gone in
jawata-mcp v3.3.1. A test pins its absence so it cannot creep back.

`/memorize` is unchanged, and the seat commands are unaffected.
