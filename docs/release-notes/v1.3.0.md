# goja-studio v1.3.0

Adds an **escape valve** to the PreToolUse enforcement guard — the first new
enforcement mechanism since the hook shipped, and the foundation for the
"try-goja-first" gate coming in a later release.

## New — declared fallback (the escape valve)

Until now, a Java symbol search over `.java` was hard-blocked with **no way
through**: if GOJA genuinely couldn't (or needn't) answer a particular search, the
agent was stuck, and the "down" message promised a re-run escape that didn't exist.

v1.3.0 makes that promise real. A search that carries an explicit **declared
fallback** proceeds — and is **logged**:

```bash
grep -n "some literal string" build.gradle   # goja-fallback: not a symbol, scanning a build file
```

Any command containing `goja-fallback: <reason>` passes the guard and appends a line
to `~/.claude/goja-studio/fallback.log` (timestamp + reason). Why this matters:

- **Fixes the footgun** — no more hard-stuck agent when grep is genuinely the right
  tool; and the down-branch message now points at a mechanism that actually works.
- **Turns a silent skip into a declared decision** — the friction that defeats
  laziness. Falling back is now a deliberate, auditable act, not a quiet default.
- **Foundation for the try-first gate** — a later release will let a fallback
  through only *after* GOJA was tried; this escape grammar is the primitive that
  gate is built on.

The escape works whether GOJA is up or down. The Grep *tool* has no free field for a
marker, so falling back means using **Bash grep with the marker** — deliberately.
Genuine symbol searches over `.java` are still redirected to GOJA's compiler-accurate
tools, health-gated as before.
